1. 规则

1.1 最左匹配原则

MongoDB 中复合索引的使用和 MySQL 中复合索引的使用类似，也有最左匹配原则。即最左优先，在检索数据时从复合索引的最左边开始匹配。

复合索引创建的时候有一个一个基本的原则就是将选择性最强的列放到最前面。

选择性最高值得是数据的重复值最少，因为区分度高的列能够很容易过滤掉很多的数据。组合索引中第一次能够过滤掉很多的数据，后面的索引查询的数据范围就小了很多了。
